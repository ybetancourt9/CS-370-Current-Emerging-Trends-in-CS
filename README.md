# CS-370-Current-Emerging-Trends-in-CS

**Briefly explain the work that you did on this project: What code were you given? What code did you create yourself?**
This project involved building the intelligent agent for a treasure hunt game where a pirate navigates a maze to find treasure before the player does. I was given the overall notebook structure, the TreasureMaze class which defines the maze environment as an 8x8 matrix and handles movement and rewards, the GameExperience class which manages the replay memory the agent learns from, the build_model function which constructs the neural network, the train_step function which performs a single gradient update, and helper functions like show, play_game, and completion_check. My contribution was implementing the qtrain function, which is the core deep Q-learning training loop. This included setting up the episode loop with epsilon-greedy action selection so the agent balances exploration and exploitation, storing experiences in replay memory, pulling batches to train the model each step, periodically syncing a target network to stabilize training, tracking the win rate over a rolling window, and adding an early stop condition that only triggers when the model both hits a near-perfect win rate and passes the completion check from every possible starting cell.

**What do computer scientists do and why does it matter?**
Computer scientists solve problems by designing systems that can process information, recognize patterns, and make decisions at a scale and speed that humans cannot. In this project that meant building an agent that teaches itself to navigate a maze through thousands of trial and error episodes rather than following hand-written rules. That same underlying approach, using data and feedback loops to produce intelligent behavior, is what powers recommendation systems, fraud detection, medical diagnostics, and autonomous vehicles. It matters because these systems increasingly make or inform decisions that affect real people, which raises the stakes on getting them right.

**How do I approach a problem as a computer scientist?**
I start by understanding what the system already does before touching anything. In this project that meant reading through TreasureMaze and GameExperience carefully to understand how state was represented, how rewards were structured, and what each method expected before I wrote a single line of qtrain. From there I worked incrementally, getting the basic episode loop running first and adding complexity like target network syncing and epsilon decay after the core logic was stable. When something didn't work I isolated the variable rather than changing multiple things at once.

**What are my ethical responsibilities to the end user and the organization?**
For the end user the main responsibility is honesty about what the system can and cannot do. A model that hits a high win rate during training but fails the completion check is not actually solved, and presenting it as working would be misleading. In a real application that kind of gap between apparent performance and actual reliability can cause real harm. For the organization the responsibility is transparency in how the model was trained, what data it used, and where it might fail. AI systems carry the biases and limitations of their design choices, and it falls on the developer to document those honestly rather than overpromising on what the system delivers.
